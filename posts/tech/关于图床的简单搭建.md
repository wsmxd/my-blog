# 图床的搭建流程

> 前置条件我们需要安装一个node，或者bun（应该可以，我没试过）来运行前端代码
- 有一些图片
- 使用cloudflare的R2存储服务（免费版有10GB）
- 编写前端代码

## 创建cloudflare端的服务
我这里用的是 Cloudflare 的 `wrangler` 命令行工具来管理 Worker 和 R2。最开始先登录一次，然后创建两个 bucket，一个给图片，一个给视频。

```bash
cd worker
pnpm install
pnpm exec wrangler login
pnpm exec wrangler r2 bucket create image
pnpm exec wrangler r2 bucket create video
```

如果你想先把本地配置文件生成出来，也可以直接跑仓库里自带的初始化脚本：

```bash
pnpm --dir worker setup
```

这个脚本会生成 `worker/wrangler.local.toml` 和可选的 `worker/.dev.vars`，里面会把这些东西都写好：

- Worker 名称
- 路由 `wsmxd.top/upload-worker/*`
- `IMAGE_BUCKET` 和 `VIDEO_BUCKET` 的绑定
- `ALLOWED_ORIGINS`、`PUBLIC_BASE_URL`、`IMAGE_PUBLIC_BASE_URL`、`VIDEO_PUBLIC_BASE_URL`
- 最大上传大小 `MAX_UPLOAD_MB`

如果要给线上环境设置上传令牌，可以再补一条：

```bash
pnpm exec wrangler secret put UPLOAD_TOKEN
```

最后本地调试和正式部署分别是：

```bash
pnpm --dir worker dev
pnpm --dir worker deploy
```

我现在的 Worker 里，图片和视频是分开绑定的：图片会进 `IMAGE_BUCKET`，视频会进 `VIDEO_BUCKET`，而公开访问地址则由 `IMAGE_PUBLIC_BASE_URL` 和 `VIDEO_PUBLIC_BASE_URL` 控制。

## 布局的策略

我这里使用的是根据不同的平宽来展示不同的列数，对于比较高的图片和比较宽的图片都采用了软压缩的方式来进行视觉效果的调整，不过我刚刚看到`pinterest`网站有一些新的地方，就是对于宽度比较大的图片采用的是旋转90度来进行展示，这样相当于使其变为了竖屏易于图床展示的样子，我现在也在考虑要不要给我的图床也换成这个效果。

## 代码的编写

我的项目已开始使用的是`vercel`的`blob`存储, 后来才使用`cloudflare`的`R2`所以我的代码采用了降级的策略，如果R2上传失败就会走到blob那里，为了处理历史的一些问题，我当时还做了不少改动，如果一开始就使用一个对象存储服务的话会简单不少。
